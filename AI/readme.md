# 幻觉

## 为什么出现幻觉

```
模型的目标是“生成最像正确答案的文本”，而不是“保证真实”
```

## 幻觉类型

1. 编造事实
2. 错误事实
3. 逻辑幻觉
4. 引用幻觉（RAG常见）

## 什么时候最容易幻觉

1. 没见过的问题 -- 冷门知识 / 新信息
2. 上下文不够 -- RAG 没检索到
3. prompt 不严格 -- 模型自由发挥
4. 问题模糊 -- “那个东西怎么样？”

## 如何减少幻觉

1. RAG -- 外部知识 → 限制模型
2. 严格 Prompt -- 基于以下上下文，回答问题。如果上下文中没有相关信息，请说"我无法从提供的上下文中找到相关信息"。
3. 引用来源 -- 回答时必须引用context
4. 降低温度（temperature） -- 更保守，不乱编 temperature = 0
5. 多模型校验 -- 一个模型答 → 另一个模型检查

# 提示词工程

## 提示词要素

1. 指令：想要模型执行的特定任务或指令。
2. 上下文：包含外部信息或额外的上下文信息，引导语言模型更好地响应。
3. 约束, 防止幻觉
4. 输入数据：用户输入的内容或问题。
5. 输出指示：指定输出的类型或格式。

# 多步骤推理（Multi-step Reasoning）

```
把一个复杂问题，拆成多个步骤，一步一步推导出答案
```

## 典型例子

```
小明有3个苹果，又买了5个，吃了2个，还剩多少？

3 + 5 = 8
8 - 2 = 6
答案：6
```

## 最经典技巧：Chain-of-Thought（CoT）

```
在 prompt 里加一句：请一步一步思考，然后给出答案
```

## Self-Consistency（多路径推理）

```
同一个问题 → 多次推理 → 投票
```

## Tree of Thought（树状推理）

```
多个思路分支 → 选择最优路径
```

## ReAct（推理 + 行动）

```
思考 → 调工具 → 再思考 → 再行动
```

## 例子

```
这个产品比去年贵多少？


1. 找去年价格（RAG）
2. 找今年价格（RAG）
3. 做差值（计算）
4. 输出结果
```

```

```

```

# 召回

召回的意思是从一堆数据里面找出最相关的

## 向量召回

```
优点：
- 语义理解强
缺点：
- 精准匹配差
```

## 关键词召回（Retrieval）

```
优点：
- 精准匹配强（比如数字、ID）
缺点：
- 不懂语义
```

```

```

```

```

- 时间范围
- 类别
```
