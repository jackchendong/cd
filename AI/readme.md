# 幻觉

## 为什么出现幻觉

```
模型的目标是“生成最像正确答案的文本”，而不是“保证真实”
```

## 幻觉类型

1. 编造事实
2. 错误事实
3. 逻辑幻觉
4. 引用幻觉（RAG常见）

## 什么时候最容易幻觉

1. 没见过的问题 -- 冷门知识 / 新信息
2. 上下文不够 -- RAG 没检索到
3. prompt 不严格 -- 模型自由发挥
4. 问题模糊 -- “那个东西怎么样？”

## 如何减少幻觉

1. RAG -- 外部知识 → 限制模型
2. 严格 Prompt -- 基于以下上下文，回答问题。如果上下文中没有相关信息，请说"我无法从提供的上下文中找到相关信息"。
3. 引用来源 -- 回答时必须引用context
4. 降低温度（temperature） -- 更保守，不乱编 temperature = 0
5. 多模型校验 -- 一个模型答 → 另一个模型检查
